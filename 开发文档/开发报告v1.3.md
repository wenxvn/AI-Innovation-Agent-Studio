## 一、项目基本信息总结

| 项目项 | 当前情况 |
|---|---|
| 项目名称 | AI Innovation Agent Studio / 智创工坊 / Idea2MVP Agent Studio |
| 前端框架 | Next.js 15 + React 19 + TypeScript |
| 后端框架 | FastAPI + SQLAlchemy + Pydantic |
| 数据库 | PostgreSQL 16 |
| 向量数据库/向量扩展 | pgvector (Vector 1536 维) |
| Redis/队列 | Redis 7 (配置存在，未实际使用) |
| 文件存储 | 本地 uploads/ 目录 |
| UI 组件库 | shadcn/ui + Tailwind CSS + Radix UI |
| Agent 框架 | 自研 Agent Runtime (无 LangGraph) |
| RAG 是否实现 | ✅ 是 (文档上传→解析→分块→Embedding→向量检索) |
| Memory 是否实现 | ✅ 是 (关键词匹配检索) |
| Skill Registry 是否实现 | ✅ 是 (硬编码 Skill 定义) |
| Tool Calling 是否实现 | ✅ 是 (硬编码工具注册表 + Tool Call 记录) |
| Eval 是否实现 | ✅ 是 (规则评估 + LLM Judge) |
| Trace/Observability 是否实现 | ✅ 是 (TraceEvent 记录) |
| 当前是否主要依赖 mock data | ⚠️ 部分依赖 (未配置 LLM API Key 时使用 Mock Provider) |

**额外说明：**

- 当前项目是 **monorepo** 结构，使用 pnpm workspace。
- 目录结构包含 `apps/web`（前端）、`apps/api`（后端）、`skills/`（Skill 定义）、`evals/`（Eval 配置）。
- 当前项目可被视为 **"可用 MVP"**，具备真实前后端、数据库、Agent Runtime、RAG、Memory、Eval、Trace 等核心能力，但部分功能依赖 Mock Provider。
