## 一、项目基本信息总结

| 项目项 | 当前情况 |
|---|---|
| 项目名称 | AI Innovation Agent Studio（智创工坊） |
| 前端框架 | Next.js 15 + React 19 + TypeScript |
| 后端框架 | FastAPI (Python 3.11+) |
| 数据库 | PostgreSQL 16 + pgvector 扩展 |
| 向量数据库/向量扩展 | pgvector（原生集成在 PostgreSQL 中） |
| Redis/队列 | Redis 7（已配置，尚未在业务逻辑中使用） |
| 文件存储 | 本地文件系统（`uploads/` 目录），MinIO 已配置但未接入 |
| UI 组件库 | shadcn/ui（基于 Radix UI） + Tailwind CSS + Lucide Icons |
| Agent 框架 | 自研 Agent 编排（非 LangGraph，基于关键词匹配的 Skill 路由 + LLM Provider 抽象） |
| RAG 是否实现 | **已实现**（文档上传 → 解析 → 分块 → Embedding → pgvector 向量检索） |
| Memory 是否实现 | **已实现**（四层类型：project/user/semantic/experience，关键词匹配检索） |
| Skill Registry 是否实现 | **已实现**（12 个内置 Skill，支持从磁盘 YAML 加载、启用/禁用） |
| Tool Calling 是否实现 | **部分实现**（Tool 注册表 + 审批机制已实现，但实际工具调用逻辑尚未接通） |
| Eval 是否实现 | **已实现**（规则评估 + LLM-as-Judge 双模式，自动在 Agent Run 后执行） |
| Trace/Observability 是否实现 | **已实现**（TraceEvent 表 + API，每个 Agent Run 自动记录 trace 事件） |
| 当前是否主要依赖 mock data | **否**。后端连接真实 PostgreSQL 数据库，前端通过 API 获取真实数据。当未配置 LLM API Key 时，Agent 输出回退到 MockLLMProvider 返回预置模板文案 |

### 附加说明

- **Monorepo**：是。使用 pnpm workspace，配置了 `apps/web` 和 `apps/api` 两个子项目，`pnpm-workspace.yaml` 存在且配置正确。
- **目录结构**：有 `apps/web`、`apps/api` 目录。`skills-main` 目录是外部 Skill 系统（包含 skill-creator、pdf、docx、xlsx 等 Skill），暂未直接集成到项目中。无独立 `evals/` 和 `packages/` 目录。
- **工程级评估**：当前项目**不只是静态 UI 原型**。它是一个具有真实后端 API、真实数据库、真实 Agent 运行链路（含 RAG、Memory、Eval、Trace）的半成品 MVP。前端所有页面均通过 React Query 连接后端 API，数据持久化在 PostgreSQL 中。
